动态规划总结

动态规划求解的一般思路：

  1. 判断问题的子结构，当具有最优子结构时，动态规划可能适用
  2. 求解重叠子问题，一个递归算法不断地调用同一问题，递归可以转化为查表从而利用子问题的解。
  3. 重新构造一个最优解

1.硬币找零

难度评级：★

　　假设有几种硬币，如1、3、5，并且数量无限。请找出能够组成某个数目的找零所使用最少的硬币数。

2.字符串相似度/编辑距离（edit distance）

难度评级：★

　　对于序列S和T，它们之间距离定义为：对二者其一进行几次以下的操作(1)删去一个字符；(2)插入一个字符；(3)改变一个字符。每进行一次操作，计数增加1。将S和T变为同一个字符串的最小计数即为它们的距离。给出相应算法。

3.最长公共子序列(Longest Common Subsequence,lcs)

难度评级：★

　　对于序列S和T，求它们的最长公共子序列。例如X={A,B,C,B,D,A,B}，Y={B,D,C,A,B,A}则它们的lcs是{B,C,B,A}和{B,D,A,B}。求出一个即可。

4.最长递增子序列（Longest Increasing Subsequence,lis）

难度评级：★

　　对于一个序列如1，-1，2，-3，4，-5，6，-7，其最长第增子序列为1,2,4,6。

5.最大连续子序列和/积

难度评级：★

　　输入是具有n个数的向量x，输出时输入向量的任何连续子向量的最大和。

6.矩阵链乘法

难度评级：★

　　一个给定的矩阵序列A1A2…An计算连乘乘积，有不同的结合方法，并且在结合时，矩阵的相对位置不能改变，只能相邻结合。根据矩阵乘法的公式，10100和1005的矩阵相乘需要做101005次标量乘法。那么对于维数分别为10100、1005、550的矩阵A、B、C，用(AB)C来计算需要101005 + 105500 =7500次标量乘法；而A(BC)则需要100550+10100*50=75000次标量乘法。

　　那么对于由n个矩阵构成的链，对i-1,2,…n，矩阵Ai的维数为pi-1*pi，对乘积A1A2…An求出最小化标量乘法的加括号方案。

7.0-1背包

难度评级：★★

　　一个贼在偷窃一家商店时发现了n件物品，其中第i件值vi元，重wi磅。他希望偷走的东西总和越值钱越好，但是他的背包只能放下W磅。请求解如何放能偷走最大价值的物品，这里vi、wi、W都是整数。

8.有代价的最短路径

难度评级：★★★

　　无向图G中有N个顶点，并通过一些边相连接，边的权值均为正数。初始时你身上有M元，当走过i点时，需要支付S(i)元，如果支付不起表示不能通过。请找出顶点1到顶点N的最短路径。如果不存在则返回一个特殊值，如果存在多条则返回最廉价的一条。限制条件：1<N<=100; 0<=M<=100 ; 对任意i, 0<=S[i]<=100。

9.瓷砖覆盖（状态压缩DP）

难度评级：★★★

　　用 1 2 的瓷砖覆盖 n m 的地板，问共有多少种覆盖方式？

10.工作量划分

难度评级：★★

　　假设书架上一共有9本书，每本书各有一定的页数，分配3个人来进行阅读。为了便于管理，分配时，各书要求保持连续，比如第1、2、3本书分配给第1人，4、5分配给第二人，6，7，8，9分配给第3人，但不能1，4，2分配给第1人，3，5，6分配给第2人。即用两个隔板插入8个空隙中将9本书分成3部分，书不能换位。同时，分配时必须整本分配，同一本书不能拆成两部分分给两个人。为了公平起见，需要将工作量最大的那一部分最小化，请设计分配方案。用s1,…,sn表示各本书的页数。

11.三次捡苹果

难度评级：★★★

　　（问题1的相关问题(1)的进一步扩展）一个矩形区域被划分为N*M个小矩形格子，在格子(i,j)中有A[i][j]个苹果。现在从左上角的格子(1,1)出发，要求每次只能向右走一步或向下走一步，每经过一个格子就把其中的苹果全部拿走，最后到达(N,M)。此时，只允许向上或向左走一步，反方向走回(1,1)。这一趟可以不走第一趟的路线，但当经过第一趟所经过的格子时，里面已经没有苹果了。到达(1,1)后，再次反方向地只允许向右或向下走，走到(N,M)，同样可以不走前两趟走过的路线。求这三趟的走法，使得最终能拿取最多数量的苹果。

二分查找问题总结

Search Insert Position

之所以会返回lo，因为如果跳出循环，必然存在lo > hi，而target大于此时lo的前一个值，则必然是比前一个lo更大的值，结合二分查找，即现在的lo。

Find First and Last Position of Element in Sorted Array

这个题目还是很清晰的；利用二分查找，如果target不在数组中，则first和last的值不会改变仍旧是-1，最后返回即可。如果，在数组中找到该值，则首先为first和last都赋值mid；然后将first往前移，last往后移动，直到它们指向的值不等于target，此时得到结果，可以break。

Search in Rotated Array

这是二分查找的变形题目，从题意来看，是将原本有序的数组从某一处截断然后旋转，最终数组是两个部分分别有序，且后一部分都小于前一部分。在这种情形下，仍旧可以应用二分查找。

  1. 首先判断特殊情况，数组中的元素只有一个，则只会存在返回0和-1的情况
  2. 对于一般情形，先还是按照二分查找的一般套路来；其中复杂的判断式

int num = (nums[mid] < nums[0]) == (target < nums[0]) ? nums[mid] ​ : target < nums[0] ? Integer.MIN_VALUE : Integer.MAX_VALUE;

  • 如果nums[mid] < nums[0]且target < nums[0]，即此时，mid已经进入了后半部分中且target显然也位于后半部分，此时让num = nums[mid]；最后的lo, hi变换中，正常进行判断，跳过不必要的检查
  • 如果nums[mid] > nums[0]且target > nums[0]，即此时，mid进入了前半部分，且target位于前半部分，此时让num = nums[mid]；最后的lo, hi变换中，正常进行判断，跳过不必要的检查
  • 如果nums[mid] < nums[0]且target > nums[0]，即此时，mid已经进入了后半部分且target位于前半部分，让num=Integer.MAX_VALUE；最后的lo, hi变换中必然有，hi = mid, 跳过了mid之后的数组元素检查
  • 如果nums[mid] > nums[0] 且 target < nums[0]，即此时，mid已经进入了前半部分且target位于后半部分，让num=Integer.MIN_VALUE；最后的lo, hi变换中必然有，lo = mid + 1, 跳过了mid之前的数组元素检查

（以上涵盖了四种不同的情况）

总结

从最后一个变形的二分查找题目可以知道，二分查找本质上是让我们利用“有序”的特点来跳过某些不必要的检查；所以要仔细思考清楚如何利用“有序”的特点和检查结果来跳过不必要检查。（可以思考分情况讨论）

回溯问题总结

发表于 2018-11-25 | 分类于 回溯 ， algorithm ， algo-summary |

回溯问题总结

Combination Sum

由于数组中不存在重复，且可以重复使用数组中的元素，所以只要注意将该步迭代加入的元素的下标作为标记直接传递给下次迭代即可。这样就可以保证所有的迭代过程中产生的解必然是独一无二的。

Combination Sum II

相对于上一个题目，多了一个要求就是所有元素只能被使用一次，此时增加了三个操作：

  • 对数组进行排序，方便之后“去重”
  • 通过判断是否是重复元素，跳过不必要的添加
  • 将迭代参数的i换成了i + 1保证了所有元素只会被使用一次

如果使用标记数组仍旧会存在重复使用的情形，只有通过数组排序后，检查当前元素是否和上一次元素一致来跳过迭代。但同时，重复元素也还是保证了能够被同时用到，因为经过了排序，所以如果是连续取用重复元素是可以的，同时也可以保证取完；但是如果是跳跃着选元素，就会被阻止（这样也就是重复情形）

Permutations

数组中的元素是独一无二的，但是不能通过位置标记start来避免重复，因为要产生所有的排列，必然还是需要每次从头开始遍历。

如果判断是否contains的代价很高，可以利用空间换时间，用boolean[] marked来标记被访问过的元素，可以迅速得到被访问与否的判断。

  1. 位置标记start: 适合用来处理在每次迭代中遍历数组“不回头”的情形
  2. 数组标记boolean[] marked: 适合用来处理每次迭代过程中仍旧会“回头”遍历的情形

Permutations II

问题改动：数组元素存在重复，但是最后的排列不能重复；相对于上一题的改动

  1. 数组进行排序，方便之后”去重“

  2. 增加了访问数组，首先用是否访问来避免添加已经被添加过的元素，利用i > 0 && nums[i] == nums[i - 1] && !marked[i - 1]来避免重复添加和已经被添加元素相等的元素

    i > 0 && nums[i] == nums[i - 1] && !marked[i - 1]

    这个核心判断，保证了如果是直接的从一开始就添加重复元素，是允许的，如果是跳过一个再添加则被阻止，如果可以跳过一个再添加证明本身是重复添加，所以被阻止。

Subsets

数组中元素是唯一的，返回所有可能的子集合；只需要位置标记start保证不回头即可

Subsets II

题目改动：数组元素存在重复，但是子集不能存在重复，对应的策略

  1. 数组排序； 2. 仍旧是阻止”跳着“添加重复元素，只允许连续添加重复元素

Palindrome Partitioning

这里使用了位置标记start，显然是”不回头“的搜索，这样可以避免重复添加；同时这里的添加检查只是更为复杂，之前的添加检查都是单纯的重复检查，这里结合题目要求增加了是否是Palindrome Partitioning的检查；本质上和以上的题目是一致的。

Sudoku Solver

这道题目仍旧是回溯法题目，只是回溯法的问题空间从一维升高到了二维之中。首先是棋盘本身是二维的空间，在添加一个元素时，需要考虑添加1-9中的任一元素；这里通过checkValid判断添加之后棋盘是否仍旧满足数独规则来增添元素，而重复性检查通被board[i][j] == '.'所替代。

N-Queens

和数独问题十分类似，只是问题域是二维空间，多了一层嵌套循环而已，同时既有row标记的位置，又有业务检查

N-Queens II

同样是利用了回溯解法，只是利用了只用求解解的个数的特点，减少了一层循环。

总结

回溯问题无论问题域是一维空间或是二维空间都具有其规律性，和普遍的框架，即：

  1. 判断是否符合条件，加入到解集合中；
  2. 遍历问题域，构造解并进行重复性检查和业务检查
  3. 设置域，将构造解带入递归调用
  4. 回退，重置域，进入下一步迭代

在回溯过程中的重复问题，常用的解决手段有：

  1. 位置标记start：用于问题域遍历”不回头“的回溯问题
  2. 访问数组boolean[]：用于问题域遍历”回头”的回溯问题

Regular Expression Matching

一、问题描述

  • substring with concatenation of all words
  • regular expression matching
