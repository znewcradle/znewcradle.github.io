Hadoop2.9.1完全分布式环境搭建

前言

本文先介绍伪分布式集群的搭建，并以此为基础来搭建由三个节点构成的Hadoop完全分布式集群（区别于单节点伪分布式集群），最后通过WordCount案例来测试集群搭建的正确性。

搭建环境

  • CentOS7
  • Hadoop2.9.1

实验过程

一、 搭建伪分布式集群

  1. 安装JDK，并配置环境变量

    可以选择下载Oracle官方JDK，也可以使用直接使用yum安装open jdk，在安装完后需要配置环境变量。需要注意的是：

    • ~/.bashrc是局部环境变量；/etc/profile是全局环境变量
    • 配置内容如下：

  2. 安装Hadoop

    • 解压hadoop安装包到opt目录(软件安装默认安装目录)

      tar zxvf hadoop-2.9.1.tar.gz -C /opt

    • 创建软链接方便之后使用

      ln -snf /opt/hadoop-2.9.1 /opt/hadoop

    • 配置环境变量，在/etc/profile中添加

    • 利用source /etc/prifle使配置生效后，使用hadoop version命令测试是否配置成功

  3. 配置Hadoop

    配置文件存放在/opt/hadoop/etc/hadoop，主要需要修改的文件如下：

    • 在hadoop-env.sh中配置JAVA_HOME，改成绝对路径
    • core-site.xml

    • hdfs-site.xml

    • mapred-site.xml：将mapred-site.xml.template文件复制一份并更名即可

    cp mapred-site.xml.template mapred-site.xml

    • yarn-site.xml
    • 创建相关目录并递归设置权限
      • 以上配置文件中尚不存在的目录需要创建
      • 递归设置读写权限chmod -R 777 /home/hadoop
    • 对HDFS集群进行格式化：hdfs namenode -format

  4. 启动集群

    • 启动HDFS集群

    • 启动YARN集群

    • 启动作业历史服务器

      mr-jobhistory-daemon.sh start historyserver

    • 利用jps命令查看是否启动成功：jps

    • WEB监控页面

      • HDFS: http://your_ip:50070
      • YARN: http://your_ip:8888（端口默认为8088，可以在yarn-site.xml文件中的yarn.resourcemanager.webapp.address进行修改）

  5. 运行自带的Demo程序

    demo程序在/opt/hadoop/share/hadoop/mapreduce/，可以运行计算PI值的程序

    yarn jar /opt/hadoop/share/hadoop/mapreduce/hadoop-mapreduce-examples-2.9.1.jar pi 4 100

二、 搭建Hadoop全分布式集群

  1. 准备工作

    • 安装JDK并在/etc/profile中配置
    • 安装伪分布式Hadoop集群

  2. 集群SSH免密登陆设置

    • hosts文件设置：在/etc/hosts文件末尾添加IP项，使得可以自己用名字访问

    注意事项：不要在127.0.0.1后面添加主机名，如果加了master，在slave节点应该解析出master IP的时候解析出127.0.0.1，造成hadoop搭建完全正确，但是系统显示可用节点一直为0。

    • hostname修改sudo hostnamectl set-hostname master

    • SSH设置：设置master和slave节点之间的双向SSH免密通信

      • 生成master的rsa密钥：ssh-keygen -t rsa；所有设置采用默认值

      • 将生成的rsa追加写入授权文件

        cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

      • 给授权文件权限：chmod 600 ~/.ssh/authorized_keys

      • 进行本机ssh测试：ssh master正常免密登陆后所有的ssh第一次都需要密码，此后都不需要密码

      • 将master上的authorized_keys传到slave1

      sudo scp ~/.ssh/id_rsa.pub hadoop@slave1:~/

      • 登陆到slave1操作：$ssh slave1输入密码登陆

      cat ~/id_rsa.pub >> ~/.ssh/authorized_keys

      • 修改authorized_keys权限：$chmod 600 ~/.ssh/authorized_keys
      • 退出slave1：$exit
      • 进行免密ssh登陆测试：$ssh slave1

    3. 执行WordCount程序进行验证

      • 准备测试文件testWordCount

      • 创建Hadoop文件夹wordCountInput: hadoop fs -mkdir /iInput

      • 将文件testWordCount上传至wordCountInput文件夹：$hadoop fs -put testWordCount /input

      • 执行wordcount程序，并将结果放入wordCountOutput文件夹：$yarn jar /opt/hadoop/share/hadoop/mapreduce/hadoop-mapreduce-examples-2.9.1.jar wordcount /wordCountInput/testWordCount /wordCountOutput

        (输出文件夹必须是没有创建过的文件夹)

      • 查看生成文件夹下的文件：$hadoop fs -ls /wordCountOutput

      • 在wordCountOutput/part-r-00000可以看到程序执行结果：$hadoop fs -cat /wordCountOut/part-r-00000
